# CLAUDE.md

Guidance for AI agents working in this monorepo.

## Repository Overview

Gem Wallet is an open-source, multi-chain cryptocurrency wallet — native iOS and Android apps with a shared Rust core.

```
wallet/
├── ios/          # SwiftUI app (Swift, SPM, Xcode)
├── android/      # Kotlin/Compose app (Gradle, Hilt)
├── core/         # Shared Rust blockchain engine (FFI → Swift, JNI → Kotlin)
└── justfile      # Root task runner (delegates to ios/ and android/)
```

## Cross-Platform Awareness

**Both apps consume the same Rust core and share domain concepts** (chains, assets, wallets, transactions, staking, swaps, WalletConnect).

1. **Core changes affect both platforms.** After modifying `core/`, regenerate bindings and verify both apps build:
   ```bash
   just generate
   just ios build
   just android build
   ```
2. **Feature parity matters.** If a change adds or modifies user-facing behavior, check whether the other platform needs an equivalent update. Call it out if so.
3. **Shared models are generated.** Files marked "Generated by typeshare" are produced from `core/`. Never edit them directly — modify the Rust source and run `just generate`.
4. **Localization is shared.** Run `just localize` to update both platforms.

## Commands

```bash
just                    # list all commands
just build              # build both iOS and Android
just generate           # regenerate models + bindings for both platforms
just localize           # update localization for both platforms
just core-upgrade       # update core submodule to latest
```

Platform-specific:
```bash
just ios build          # build iOS only
just ios test-all       # run all iOS tests
just android build      # build Android only
just android test       # run Android tests
```

## Architecture

| Layer | iOS | Android | Shared |
|-------|-----|---------|--------|
| **UI** | SwiftUI + MVVM | Jetpack Compose + MVVM | — |
| **DI** | Environment injection | Hilt | — |
| **Data** | Store (SQLite) + Services | Room + Repositories | — |
| **Blockchain** | Gemstone (FFI) | Gemstone (JNI) | Rust `core/` |

## Task Completion

1. **Build the affected platform(s)** before claiming work is done
2. **Run tests** for affected platform(s)
3. If `core/` was touched, build and test **both** platforms
4. No dead code, no unnecessary comments, clean imports
5. Follow each platform's existing patterns (see platform guides)
